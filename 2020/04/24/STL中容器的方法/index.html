STL中容器的方法

Posted on 2020-04-24 Edited on 2020-04-26 In C++

这篇文章主要汇总STL中容器的常用方法

容器介绍

  • 顺序容器
    • 可变长动态数组vector
    • 双端队列deque
    • 双向链表list
  • 关联容器
    • set
    • multiset
    • map
    • multimap
  • 适配器
    • 栈 stack
    • 队列 queue
    • 优先队列 priority_queue

所有容器拥有的成员函数

  • size():返回容器对象中元素个数
  • empty():判断容器对象是否为空

顺序容器和关联容器都拥有的成员函数

  • begin(): 返回指向容器中的第一个迭代器
  • end() : 返回容器中最后一个元素后面的迭代器
  • rbegin()：指向容器最后一个元素的反向迭代器
  • rend():指向第一个元素前面的反向迭代器
  • erase(): 从容器中删除一个元素，或几个元素
  • clear(): 容器中删除所有元素

顺序容器都拥有的成员函数

  • front() : 返回容器中第一个元素的引用
  • back(): 返回容器最后一个元素的引用
  • push_back(): 在容器末尾插入元素
  • pop_back():删除元素末尾的元素
  • insert() : 插入一个或多个元素

vector成员函数

构造函数

vector<T>这个类中有四个构造函数的重载：

  • vector(): 无参数构造函数，将容器初始化为空
  • vector（int n）: 将容器初始有n个元素
  • vector(int n, const T& val): 容器中包含n个val
  • vactor(iterator first, iterator last): 利用迭代区间[first, last)初始化vector

insert&erase

这两个函数均有重载函数：作用于单个迭代器 + 某个迭代器区间

单个迭代器：

  • inset(iterator i , const T& val)
  • erase(iterator i)

多个迭代器

  • insert(iterator i, iterator first, iterator last)
  • erase(iterator first, iterator last)

void swap(vector<T>& v): 将容器自身的内容和另一同类型的容器内容v互换；

list

list容器和vector容器在成员函数最大的区别是：list可以在头部进行插入删除操作：

  • void push_back(const T& val): 将val插入最前面；
  • void pop_front(): 删除链表最前面的元素
  • void sort(): 将链表从小到大排序
  • void remove(const T& val): 删除和val相等的元素
  • remove_if(): 删除某些复合条件的元素
  • unique()
  • merger
  • splice
    list中没有sort函数，因为sort需要随机迭代器的支持，但是list是双向迭代器；

deuqe

所有使用与vector的操作均使用于deque，由于deque支持双端操作，因此多了以下几个成员函数：

  • void push_front(const T& val) // 将val插入容器的头部
  • void pop_front() // 删除容器的头部

multiset

定义：

set

multiset拥有的成员函数，set均有。由于set不能有重复元素，因此insert有区别；
函数原型：

假设返回的pair对象为x，如果插入成功x.second为true, x.first指向被插入元素的迭代器。如果插入失败，x.second为false，x.first指向原来那个元素的迭代器；

multimap

定义：

multimap中的元素是pair模板类对象。元素的first为关键字，second为值；其中元素按关键字大小排列；注意，map和multimap中的find和count不用==运算符比较是否相等。而是通过x<y和y<x同时为假，来认定y和x相等；

map

定义：

multimap有的成员函数，map都有。于此之外，还有operator[]:

也就是通过下标操作访问val，如果存在返回该值，如果不在添加一个first为k的值；

适配器

适配器有stack，queue，priority_queue三种。他们都是在顺序容器基础上实现的，屏蔽和加强一部分功能；
适配器都有以下三个成员函数：

  • push:添加一个元素；
  • top:返回栈顶，对头元素的引用 （修改：对于queue，应该使用front()）
  • pop: 删除一个元素
    迭代器是容器和算法之间的桥梁，由于适配器没有迭代器，因此，STL中的算法不适应容器适配器；

stack

定义：

stack默认以deque实现，也可指定用vector和list实现；

queue

定义：

queue默认deque实行，可指定list实现
push在队尾，pop在对头，而stack全部操作在栈顶；

priority_queue

定义：

  • 第二个参数：指定由什么实现，默认为vector，可指定为queue；
  • 第三个参数：元素的比较方法，默认为大堆。可指定为小堆，greater
